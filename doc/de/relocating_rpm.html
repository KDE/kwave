2.6. Installation des Binary-RPM-Paketes auf anderen Distributionen

Das Binary-rpm-Paket von Kwave ist "verschiebbar". Das heißt, dass sie das Paket für jedes System mit gesetztem Standard-KDEDIR erzeugen und es auf jedem anderen System oder in einem anderen Verzeichnis installieren können. Diese Eigenschaft ist sehr nützlich um ein für SuSE compiliertes Paket auf einem RedHat System zu installieren und umgekehrt.

Zum Beispiel: Sie haben ein Binary-rpm von einem Freund mit einem RedHat System bekommen (wo KDEDIR /usr ist) und wollen es auf einem SuSE System installieren (wo KDEDIR /opt/kde ist). Dann geben sie einfach ein unterschiedliches Präfix bei der Installtion des rpm an. Beispiel: 
 % rpm -i --prefix=/opt/kde kwave-0.5.2-9.i586.rpm

Dies modifiziert den Pfad in dem das Programm installiert wird und es wird dringend empfohlen dass dies das gleiche Verzeichnis ist, wie es in der KDEDIR Umgebungsvariable spezifiziert ist. Ansonsten ist Kwave nicht in der Lage seine Menü-Konfiguration, Plugins, Standard-Dateien und so weiter zu finden.

Das folgende sollte immer funktionieren und das gewünschte Paket an den korrekten Platz installieren - so lange die KDEDIR-Umgebungsvariable richtig gesetzt ist: 
 % rpm -i --prefix=${KDEDIR} kwave-0.5.2-9.i386.rpm
