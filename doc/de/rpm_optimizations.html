2.7. RPM-Optimierungen

Ja, sie können das RPM Paket für Pentium und Pentium Pro (oder einige andere CPUs) compilieren indem sie einige Definitionen in ihrer rpmrc Datei benutzen. Sie können auch ihre systemweite /etc/rpmrc Datei oder die .rpmrc Datei in ihrem Home-Verzeichnis verändern. Dort können sie zum Beispiel folgende Options-Zeilen angeben: 
 % optflags: i386 -O2 -m486 -DNDEBUG
 % optflags: i586 -O2 -march=pentium -DNDEBUG -fomit-frame-pointer
 % optflags: i686 -O2 -march=pentiumpro -DNDEBUG -fomit-frame-pointer
(Ich fand diese schönen Tricks auf http://www.keywarrior.net/duesti/rpmopt.de.html. Dank an Matthias Düsterhöft!).

Das bedeutet, dass auf einer i586-Architektur (Pentium) das rpm Paket unter Benutzung von "-march=pentium" als Compiler Option compiliert wird. Sie sollten dies für ihren eigenen Bedürfnisse erweitern oder anpassen. Die Einstellungen sind für alle vorhergehenden Sektionen anwendbar, wo ein "rpm -ta" benutzt wurde oder ein binary RPM mittels eines Source RPM unter Benutzung von "rpm --rebuild" erstellt wurde.

Die hier spezifizierten Optionen werden während der Compilierung in die Umgebungsvariable RPM_OPT_FLAGS geschrieben. Falls sie also keine rpm-basierte Umgebung haben und kompilieren müssen wie in Kaptitel Compilierung und Installation beschrieben, können sie diese Variable "per Hand" setzen und erzielen den gleichen Effekt. Zum Beispiel: 
 % export RPM_OPT_FLAGS="-O2 -march=pentium -fomit-frame-pointer"
