2.6. Relocating the binary RPM packages

The binary rpm package of Kwave is "relocatable". That means that you can build the package any system with KDEDIR set to some standard location and install it on an other system or into a different directory. This feature is very useful to install a package compiled for SuSE under a RedHat system or vice-versa.

For example: you got the binary rpm from a friend who has a RedHat system (where KDEDIR is /usr) and want to install it on a SuSE system (where KDEDIR is /opt/kde). Then just give a different prefix when installing the rpm. Example: 
 % rpm -i --prefix=/opt/kde kwave-0.5.2-9.i586.rpm

This modifies the path where the program is installed and it is strongly recommended that this is the same directory as specified in the KDEDIR environment variable, otherwise Kwave would be unable to find it's menu configuration, plugins, preset files and so on...

The following should always work and install the desired package in the correct place - as long as the KDEDIR environment variable is correctly set: 
 % rpm -i --prefix=${KDEDIR} kwave-0.5.2-9.i386.rpm
